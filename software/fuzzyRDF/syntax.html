The syntax is pretty easy and nothing else than RDF. Examples can be found in the "testRDFXMLdir" folder of the download.

An example RDFS encoding of fuzzy RDFS statements

< (SportsCar, subclassOf , FastCar), 0.9 >

< (FastCar, subclassOf , ExpensiveCar), 0.7 >

< (ExpensiveCar, subclassOf , PassengerCar), 0.3 >

is the following:

<?xml version="1.0"?>
<!DOCTYPE rdf:RDF [<!ENTITY xsd "http://www.w3.org/2001/XMLSchema#">]>
<rdf:RDF xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
xmlns:rdfs="http://www.w3.org/2000/01/rdf-schema#"
xmlns:fuzzy="http://gaia.isti.cnr.it/~straccia#"
xml:base="http://gaia.isti.cnr.it/~straccia/base#"
>

<rdf:Statement rdf:about="#triple1">
<rdf:subject rdf:resource="#SportsCar"/>
<rdf:predicate rdf:resource="http://www.w3.org/2000/01/rdf-schema#subClassOf" />
<rdf:object rdf:resource="#FastCar" />
<fuzzy:degree rdf:datatype="&xsd;decimal">0.9</fuzzy:degree>
</rdf:Statement>

<rdf:Statement rdf:about="#triple2">
<rdf:subject rdf:resource="#FastCar"/>
<rdf:predicate rdf:resource="http://www.w3.org/2000/01/rdf-schema#subClassOf" />
<rdf:object rdf:resource="#ExpensiveCar" />
<fuzzy:degree rdf:datatype="&xsd;decimal">0.7</fuzzy:degree>
</rdf:Statement>

<rdf:Statement rdf:about="#triple3">
<rdf:subject rdf:resource="#ExpensiveCar"/>
<rdf:predicate rdf:resource="http://www.w3.org/2000/01/rdf-schema#subClassOf" />
<rdf:object rdf:resource="#PassengerCar" />
<fuzzy:degree rdf:datatype="&xsd;decimal">0.3</fuzzy:degree>
</rdf:Statement>

</rdf:RDF>

Query Language:

A query expression is a conjunctive query of the following form.

Variables and variable lists are defined as:

<variableList> ::= <variable>
| <variable> <variableList>
<relationList> ::= stm(<?>, <?>, <?>)
| stm(<?>, <?>, <?>) <relationList>
| stm(<?>, <?>, <?>)[<?>]
| stm(<?>, <?>, <?>)[<?>] <relationList>
<?> ::= <variable>
| <identifier>
| <costantString>

A query may be of the form:

(RETRIEVE (<variableList>) WHERE (<RelationList>))

For instance, a simple query is (here ?Z will contain the score)

RETREIVE (?X ?W ?Z) WHERE ((?X subclassof ?W)[?Z])

A query may be of the form:

(RETRIEVE (<variableList>) WHERE (<RelationList>) ORDERBY (<function> AS <identifier>) LIMIT (k))

For instance, a complex query is

RETREIVE (?X ?W) WHERE ((?X subclassof ?W)[?Z]) ORDERBY ( rs(?Z,0,0.5) AS score) LIMIT 10

  • "find top-10 ranked tuples ranked according to the score calculated as a right-hand shoulder RS(0,0.0.5) using the value of ?Z"

Functions are the right-shoulder, left-shoulde, triangular and trapezoidal functions and are defined as:

<function> ::= rs(<variable>, <numericConstant>, <numericConstant>)
| ls(<variable>, <numericConstant>, <numericConstant>)
| tnf(<variable>, <numericConstant>, <numericConstant>, <numericConstant>
| trf(<variable>, <numericConstant>, <numericConstant>, <numericConstant>

A query may be of the form:

(RETRIEVE (<variableList1>) WHERE (<RelationList>) GROUPBY (<variableList2>) ORDERBY (<aggregate>AS <identifier>) LIMIT (k))

which is the previous case, except that now we allow grouping according to an aggregate function, that may be:

<aggregate> ::= max(<function>)
| min(<function>)
| sum(<function>)
| avg(<function>)

Note: Each variable occurring in the variable list has to occur in at least one relation in the relation list 2. Each variable occurring the the scoring function has to occur in at least one relation in the relation list.
