Software developed by Umberto Straccia

Logic

  • FuzzyDL This is a fuzzy Description Logic reasoner.
  • SoftFacts Ontology Mediated Access to Databases. Supports also top-k retrieval
  • FuzzyDL-Learner A tool to learn class descriptions from OWL 2 ontologies
  • fuzzyRDF Fuzzy RDF. Supports also top-k retrieval
  • DLMedia An Ontology Mediated Multimedia Information Retrieval System
  • oMap: A probabilisitc Ontology Alignemt Tool
  • State-Solver-Template: This is a TEMPLATE for a trivial search algorithm in a space of states. Each state encodes a problem to be solved. It starts with a set of states to be solved and returns true if all of them can be solved.
  • sat-L: The system presents a list of functionalities about propositional logic. testing code is provided. The system is a very naive implementation.
  • sat-UP: As sat-L, except that Unit-propagation is provided.
  • sat-S: The system presents a list of functionalities about propositional logic.
    It implements a signed tableaux. Moreover, it allows reasoning in a four-valued semantics framework (see bibitem ). Testing code is provided.
  • sat-S-UP: As sat-S, except that Unit-propagation is provided.
  • sat-F: The system presents a list of functionalities about fuzzy propositional logic. testing code is provided, too.
  • alc: The system presents a list of functionalities about the Description Logic ALC. Testing code is provided.
  • alc-F: The system presents a list of functionalities about the Fuzzy Description Logic ALC. Testing code is provided. See also bibitem .
  • Multivalued-prolog: The system presents a simple and effective way in order to manage [0,1] truth graded atoms through "Multivalued Logic Programming". A standard prolog engine is enough. You just need to load a small file into your standard prolog.
